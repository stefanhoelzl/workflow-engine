## 1. RunSequencer module (main-side, isolated)

- [x] 1.1 Create `packages/sandbox/src/run-sequencer.ts` exporting `createRunSequencer(): RunSequencer` factory with `start()`, `next(wireEvent): SandboxEvent | null`, `finish(opts?: { closeReason: string }): SandboxEvent[]`. State: `seq=0`, `refStack: number[]=[]`, `callMap: Map<number, {openSeq, name, kind}>=new Map()`, `runActive=false`, `lastSeenTs`/`lastSeenAt` for synthesis.
- [x] 1.2 Implement type-discriminated stamping in `next`: branch on the wire event's `type` field (`"leaf"` / `{ open: callId }` / `{ close: callId }`). For `"leaf"`: `ref = refStack.at(-1) ?? null`, no mutation. For `{ open: callId }`: `ref = refStack.at(-1) ?? null`, push assignedSeq onto refStack, register `callMap[callId] = {openSeq, name, kind}`. For `{ close: callId }`: lookup callMap; if absent, log `sandbox.close_without_open` and return null; otherwise `ref = openSeq`, remove openSeq from refStack, delete callMap entry. The `kind` string SHALL NOT be parsed for framing.
- [x] 1.3 Implement `finish` two paths: with `closeReason` synthesise LIFO close events using captured name/kind and `lastSeenTs`/`lastSeenAt`. Synthesis kind derivation: take the prefix before the first `.` of the captured open kind, append `.error` (e.g. `"system.request"` → `"system.error"`, `"trigger.request"` → `"trigger.error"`, `"weird-no-dot"` → `"weird-no-dot.error"`). This is failsafe — works regardless of suffix-convention adherence. Without `closeReason`: log `sandbox.dangling_frame` warning if frames open. Both paths zero state and set `runActive=false`.
- [x] 1.4 Write `packages/sandbox/src/run-sequencer.test.ts` covering: start/next/finish happy path; concurrent open/close pairing via callId (Promise.all-shape); close-without-callId logs and drops; finish-with-reason synthesises LIFO; finish-without-reason warns on dangling; out-of-window event paths surfaced via injectable logger; non-conventional kinds (e.g. `kind="user.click"` with `type:{open:n}`) frame correctly because framing reads `type`, not `kind`.
- [x] 1.5 Export `RunSequencer` type and `createRunSequencer` factory from the sandbox package internal module graph (NOT public — never re-exported from `@workflow-engine/sandbox` index).

## 3. Sandbox-context simplification
