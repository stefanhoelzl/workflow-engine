- **Upload pipeline (post-Phase-4 correction)**: restored for v1. `POST /api/workflows` accepts a gzip tarball containing `manifest.json` + `<name>.js`, extracts it, validates the manifest against `ManifestSchema`, and registers (or replaces) the workflow via `registerFromFiles()`. `WORKFLOWS_DIR` is empty by default — the runtime starts with no workflows; they arrive via upload. Mounted at `/trigger/*`, the page lists every registered HTTP trigger and renders a body-schema form per trigger (jedison on the client). Submissions POST directly to the public `/webhooks/<path>` ingress, keeping the UI a thin wrapper around the webhook plane.
